# ML KEM

ML KEM is a Quantum Resistant key exchange algorithm.

The principle of the key exchange itself is the same as with RSA key exchange:

1. Alice generates public and private key
2. Alice exposes her public key to Bob
3. Bob encrypts the shared key with Alice's public key
4. Alice decrypts the shared key with her private key

However, RSA is built on a problem known as the prime factorization of integers. This problem becomes solvable with Shor's algorithm on quantum computers

ML KEM follows similar structure as key exchange algorithm, but the problem it's built upon Learning With Errors

# Algorithm for simple LWE

This is introduction, using simple LWE. Real ML KEM uses a different variant of this problem (Although the structure generally remains the same)

Algorithm has 3 parts

1. KeyGen
2. Encapsulation
3. Decapsulation

## KeyGen

We determine the following constants

`q` = The modulus - All numbers are in range `0...q`

`n` = The dimensions - The dimensions of the matrixes and vectors

`k` = The small range - Some parts (Like the errors) need small numbers in range `-k...k`

For ML KEM the constants are

`q = 3329`

`n = 256`

`k = 3`

Alice generates the following, filled with random numbers mod q

- Public matrix `A` of dimensions `n X n` and numbers `0...q-1`
- Secret vector `s` of order `n` and numbers `-k...k`
- Small error vector `e` of order `n` and numbers `-k...k`

Alice then computes the Public vector `t` like this:

$t = A s + e \pmod q$

Then the public key is the tuple `(A, t)`

## Encapsulation

Bob generates the following

- Ephemeral secret vector `r` of order `n` and numbers `-k...k`
- Small error vector `e1` of order `n` and numbers `-k...k`
- Small error number `e2` from range `-k...k`

He also encodes the message `m`

### Encoding

Our simple LWE can only encrypt one bit at a time (this is its main disadvantage)

We encode the bit into `m` as follows

- If the bit is 0, we assign it `m = 0`
- If the bit is 1, we assign it the value `m = q/2` (Integer value)

We will learn why soon enough

### Encryption

We compute the Key vector `u` like this:

$u = A^T r + e_1 \pmod q$

As you can see, it's the exact same computation that Alice performed. There is one **big** difference however. The `T` in superscript means the matrix is 'transposed' and is vital for the computation.

And then we also compute the Ciphertext `v`

$v = r t + e_2 + m \pmod q$

The `rt` computes the dot product between the two vectors, which is a simple integer. That's why we can only transmit 1 bit (a simple integer)

The tuple `(u, v)` is the whole ciphertext that Bob sends

## Decapsulation

Alice now computes the following:

$v - u s \pmod q$

Which yields a value that is **approximately** `0` or `q/2` and therefore the bit `0` or `1`

### Why does it work?

The answer can be found after expanding the final equation (We ommit the `(mod q)`, but it's there in each step)

$v - u s \pmod q$

Expand $v = r t + e_2 + m$

$(r t + e_2 + m) - u s$

Expand $t = A s + e$

$((A s) r + r e + e_2 + m) - u s$

Expand $u = A^T r + e_1$

$((A s) r + r e + e_2 + m) - ((A^T r)s + s e_1)$

Now remove the parentheses

$`$(A s) r + r e + e_2 + m - (A^T r) s - s e_1$`$

**Important**: $(A s) r = (A^T r) s$, because we used the transposed matrix

So after cancelling those two terms out

$r e + e_2 + m - s e_1$

We got `m` + some small `errors`, but here is the thing the sum of all those errors is always `sum < q/4` (Because the errors are chosen small: `-k...k`)

Meaning that we can determine whether the `m` value is closer to `0` or to `q/2`

Note: The value wraps around, so with `q=3329`, the value 3320 is closer to `0` than to `q/2` (Because `3320 + 9 = 0 (mod q)`, so the 'distance' is 9)

### Notes on the scheme

- The LWE scheme can only transmit 1 bit at a time making it incredibly inefficient
- The public matrix A is really big to transmit, in reality the matrix is generated from a seeded PRNG and only the `seed` is transmitted
- The small values distribution (`-k...k`) is not uniform, but typically centered around 0, meaning that the noise is smaller overall
- The *noise sum is always smaller than `q/4`* is actually false, but the probability of it being larger than `q/4` is **very low**
